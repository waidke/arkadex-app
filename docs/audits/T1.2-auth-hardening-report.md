# Security Audit & Hardening Report — T1.2 Auth Flow

**Date:** 2026-05-15  
**Owner:** DevSecOps Lead  
**Status:** ✅ SIGNED-OFF  
**Milestone:** M1 Foundations

## 1. Executive Summary
Following the implementation of Phase B (Auth Middleware), a security audit was performed to identify and mitigate risks in the authentication lifecycle. This report documents the fixes for critical bugs and the implementation of proactive security measures (Hardening).

## 2. Findings & Remediation

| ID | Issue | Severity | Status | Remediation |
| :--- | :--- | :--- | :--- | :--- |
| **BUG-01** | Session cookie not persisting in Route Handlers | 🔴 Critical | **Fixed** | Added `setAll` to `createServerSupabase` using `cookies().set()` with read-only safety. |
| **SEC-01** | Open Redirect vulnerability in callback route | 🔴 Critical | **Fixed** | Added `startsWith('/')` validation to the `next` parameter. |
| **SEC-02** | Missing Security Headers (CSP/Permissions) | 🟡 Medium | **Fixed** | Added CSP and Permissions-Policy to `vercel.json`. |
| **SEC-03** | Insecure Admin Whitelist matching | 🟠 High | **Fixed** | Implemented `.trim().toLowerCase()` normalization in middleware. |
| **SEC-04** | Broken error redirect path | 🟠 High | **Fixed** | Redirected auth errors to `/?message=auth_error` (valid path). |

## 3. Rate Limiting Verification
The following rate limits have been verified in `supabase/config.toml` and are deemed adequate for the Personal Alpha phase:

- **Sign-in / Sign-up:** 30 requests per 5 minutes per IP.
- **Token Refresh:** 150 requests per 5 minutes per IP.
- **Email:** 2 per hour (Note: Email sign-up is disabled).

> [!IMPORTANT]
> The `skip_nonce_check = true` setting in `config.toml` is confirmed to be **local-only**. Hosted Supabase environments enforce standard OIDC security practices.

## 4. Content Security Policy (CSP) Analysis
The implemented CSP uses a balanced approach for the current development phase:
- `default-src 'self'`: Restricts sources to the same origin.
- `script-src 'self' 'unsafe-inline' 'unsafe-eval'`: Required for Next.js development and Framer Motion animations.
- `img-src 'self' data: https://assets.tcgdex.net`: Allows local images and TCGDex assets.
- `connect-src`: Authorized for Supabase API and WebSockets.

## 5. DevSecOps Sign-off
The authentication flow is now considered "Hardened" and ready for QA End-to-End testing (Phase D). All critical vulnerabilities identified in the audit have been closed.
