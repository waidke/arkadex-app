## 1. Schema Decision: Opsi B (Schema Migration)

**Decision:** Tambahkan kolom `supertype` dan `element` ke tabel `cards`.

**Justifikasi:**
- **User Experience:** Filter berdasarkan tipe kartu (Pokemon/Trainer) dan elemen (Fire/Water) adalah ekspektasi dasar pengguna TCG.
- **Data Integrity:** Menyimpan data ini di kolom terpisah memudahkan query dan agregasi dibandingkan menyimpan dalam JSON blob atau membuangnya.
- **Efficiency:** Biaya migrasi saat ini (Personal Alpha) hampir nol karena belum ada data produksi yang sensitif.

**Implementation Details:**
- Kolom: `supertype` (TEXT, nullable), `element` (TEXT, nullable).
- Migration script: `supabase/migrations/20260516003330_add_supertype_element_to_cards.sql`.

## 2. Format File Input (Canonical)

### 2.1 `cards.csv`
- **Encoding:** UTF-8 tanpa BOM.
- **Max Rows:** 500 per batch.
- **Header:** `set_code,card_number,card_name,rarity,supertype,element,image_url`

### 2.2 `sets.csv`
- **Encoding:** UTF-8 tanpa BOM.
- **Max Rows:** 50 per batch.
- **Header:** `code,name,release_date,total_cards`

## 3. Validation Rules

### 3.1 `cards.csv` Field Validation

| Field | Status | Format / Enum | Action on Invalid |
| :--- | :--- | :--- | :--- |
| **set_code** | REQUIRED | Alphanumeric, max 10 char | Reject entire batch (log error) |
| **card_number** | REQUIRED | Regex: `^\d{1,4}/\d{1,4}[a-z]?$` | Reject row |
| **card_name** | REQUIRED | Non-empty trim, max 255 char | Reject row |
| **rarity** | Optional | Enum: `C, U, R, RR, SR, SAR, SSAR, AR, IR, UR, PR` | Reject row if set, but invalid |
| **supertype** | Optional | Enum: `Pokemon, Trainer, Energy` | Reject row if set, but invalid |
| **element** | Optional | Enum: `Fire, Grass, Water, Lightning, Psychic, Fighting, Darkness, Metal, Dragon, Colorless, Fairy` | Reject row if set, but invalid |
| **image_url** | Optional | Regex: `^https://.+` | Reject row if set, but invalid |

### 3.2 `sets.csv` Field Validation

| Field | Status | Format | Action on Invalid |
| :--- | :--- | :--- | :--- |
| **code** | REQUIRED | Alphanumeric, max 10, UNIQUE | Reject if duplicate in DB or batch |
| **name** | REQUIRED | Non-empty, max 255 | Reject row |
| **release_date** | Optional | YYYY-MM-DD | Reject row if set, but invalid |
| **total_cards** | Optional | Positive Integer | Reject row if set, but invalid |

## 4. Business Logic Rules

1.  **Set Resolution Policy:** Jika `set_code` di `cards.csv` tidak ditemukan di database, **REJECT** seluruh batch untuk set tersebut. Sistem tidak melakukan auto-create set.
2.  **Processing Order:** Import `sets.csv` harus diselesaikan terlebih dahulu sebelum mengupload `cards.csv`.
3.  **Duplicate Handling:** Implementasi **Dry-run Preview Mode** wajib di Phase B. Tampilkan ringkasan (New, Overwrite, Error) sebelum user menekan tombol "Confirm Commit".
4.  **CSV Parsing:** Wajib menggunakan library **PapaParse** (Client-side) untuk menangani escape characters (koma dalam nama kartu).

## 5. KR4 Audit Protocol

**Goal:** KR4 — "Zero data-entry errors found after manual personal audit of logged cards."

### 5.1 Kelas Error
- **Critical:** Salah nama kartu, salah mapping set, data required kosong, set_id tidak valid.
- **Major:** URL gambar pecah (404), format nomor kartu salah (misal 1/190 vs 001/190).
- **Minor:** Typo kecil pada enum (misal "Rare" vs "R").

### 5.2 Pass Criteria
- **Pass:** 0 Critical Error, 0 Major Error.
- Minor error harus didokumentasikan untuk perbaikan batch berikutnya.

### 5.3 Sampling & Source
- **Sets:** 100% populasi audit.
- **Cards:** Random sampling 10% atau minimal 20 kartu per set.
- **Truth Source:** [TCGdex](https://tcgdex.net) atau Database Resmi Pokemon Card.
