# Manual Setup Guide — T1.2 Phase A: Google OAuth

- **Task**: T1.2 Phase A — Provider Config
- **Persona**: DevSecOps Lead
- **Audience**: Non-technical — follow each step sequentially
- **Date**: 2026-05-15

## Overview

Three areas require manual configuration via browser before any code changes can begin:

| # | Area | Purpose |
|---|---|---|
| 1 | Google Cloud Console | Register ArkaDex with Google to enable "Login with Google" |
| 2 | Supabase Dashboard | Enter the Google Client ID & Secret into Auth settings |
| 3 | Vercel Dashboard | Add the admin configuration variable to the environment |

---

## Part 1 — Google Cloud Console

### Step 1 — Open Google Cloud Console

Go to: https://console.cloud.google.com

Make sure you are signed in with your admin Google account.

### Step 2 — Select or create a project

In the top-left corner, click the active project name (or the **Select a project** button).

- If the ArkaDex project already exists: select it.
- If not: click **New Project** → name it `arkadex-prod` → click **Create**.

### Step 3 — Configure the OAuth Consent Screen

1. In the left menu, click **APIs & Services** → **OAuth consent screen**
2. Select **External** → click **Create**
3. Fill in the **App Information** section:
   - **App name**: `ArkaDex`
   - **User support email**: select your email from the dropdown
   - **Developer contact email**: your admin email address
4. Click **Save and Continue**
5. On the **Scopes** page: no changes needed → click **Save and Continue**
6. On the **Test users** page:
   - Click **+ Add users**
   - Enter your Gmail address
   - Click **Add** → **Save and Continue**
7. On the **Summary** page: click **Back to Dashboard**

> **Why Testing status?** The app stays in Testing mode during development. It will only be published after QA is complete (Phase D), not now.

### Step 4 — Create an OAuth Client ID

1. In the left menu, click **Credentials**
2. Click **+ Create Credentials** at the top of the page
3. Select **OAuth client ID**
4. Set **Application type** to **Web application**
5. Set **Name** to: `ArkaDex Supabase`

**Authorized JavaScript origins** (click **+ Add URI** for each line):

```
http://localhost:3000
http://127.0.0.1:3000
https://arkadex-app.vercel.app
```

**Authorized redirect URIs** — copy exactly as shown (click **+ Add URI** for each line):

```
http://127.0.0.1:54321/auth/v1/callback
https://qdtfecuzatcbggypllhr.supabase.co/auth/v1/callback
```

> **Important**: Redirect URIs point to the Supabase endpoint, not the application domain. This is correct — do not change them.

6. Click **Create**

### Step 5 — Save Credentials

After clicking Create, a popup will appear with two values:

- **Your Client ID** — long string ending in `.apps.googleusercontent.com`
- **Your Client Secret** — starts with `GOCSPX-`

> **Required**: Copy both values to your **password manager** or a secure location before closing the popup. You will need them in Part 2.

Click **OK** to close the popup.

## Part 2 — Supabase Dashboard

### Step 1 — Open the Providers page

Navigate directly to:

```
https://supabase.com/dashboard/project/qdtfecuzatcbggypllhr/auth/providers
```

Log in to Supabase if prompted.

### Step 2 — Enable the Google provider

1. Scroll down to the **Social Auth** section — find **Google**
2. Click the toggle next to **Google** until it turns green (enabled)
3. Fill in the two fields that appear:
   - **Client ID (for OAuth)**: paste the **Client ID** from Part 1, Step 5
   - **Client Secret**: paste the **Client Secret** from Part 1, Step 5
4. Click **Save**

### Step 3 — Disable Email/Password

On the same page (still in Authentication > Providers):

1. Find the **Email** section — usually at the top of the list
2. Ensure the **Enable Email Signup** toggle is **OFF** (grey/disabled)
3. If it is still ON, click the toggle to disable it → click **Save**

> **Why disable it?** ArkaDex uses Google OAuth only. Email/password authentication is not permitted per the architecture decision (ADR-003).

### Step 4 — Configure Allowed URLs

Navigate to:

```
https://supabase.com/dashboard/project/qdtfecuzatcbggypllhr/auth/url-configuration
```

1. Verify the **Site URL** field contains:
   ```
   https://arkadex-app.vercel.app
   ```

2. Under **Redirect URLs**, add the following three entries (click **+ Add URL** for each):
   ```
   https://arkadex-app.vercel.app/**
   http://localhost:3000/**
   http://127.0.0.1:3000/**
   ```

3. Click **Save**

## Part 3 — Vercel Dashboard

### Step 1 — Open Environment Variables

1. Go to: https://vercel.com/dashboard
2. Select the **arkadex-app** project
3. Click the **Settings** tab
4. In the left menu, click **Environment Variables**

### Step 2 — Add the new variable

Click **+ Add** and fill in as follows:

| Field | Value |
|---|---|
| **Key** | `ADMIN_EMAIL_WHITELIST` |
| **Value** | `<your-admin-email@gmail.com>` |
| **Environment** | Check **Production** and **Preview** only (leave Development unchecked) |

Click **Save**.

> **Note**: The Google Client ID and Secret do **not** need to be added to Vercel. They are stored directly in the Supabase Dashboard (Part 2). Vercel does not require those credentials.
